Códigos de barras

Es una representación de datos que pueden ser leídos por una máquina.

Generalmente describe información del producto que tiene el código.

## Historia
### 1952 Norman Joseph Woodland y Bernard Silver patentaron la idea el 7 de octubre llamada "Aparato y método de clasificación" Desarrollaron un sistema automático para leer información de productos a la hora de pagar Para leer los códigos adaptaron la tecnología óptica de sonido usado en las peliculas

Aparato y método de clasificación

Patente original. Tomado de https://www.google.com/patents/US2612994

### 1959 En 1959 David Collins trabajó en el Ferrocarril de Pensilvania y se dio cuenta de la necesidad de identificar automáticamente los vagones de ferrocarril. Él desarrolló un sistema llamado KarTrak usando rayas reflectantes azules y rojas unidas al lado de los coches
### 1966 En 1966 la Asociación Nacional de Cadenas Alimenticias (NAFC) comenzó a discutir la idea de los sistemas automáticos de pago. En ese momento, RCA poseía los derechos sobre la patente original de Woodland and Silver y comenzó un proyecto interno para desarrollar un sistema efectivo.
### 1972 1972, RCA comenzó una prueba de dieciocho meses en una tienda de Kroger en Cincinnati. El código demostró tener un problema serio; las impresoras a veces manchan la tinta, haciendo que el código sea ilegible en la mayoría de las orientaciones.
### 1973 Woodland en IBM, se imprimió en la dirección de las rayas, por lo que la tinta adicional simplemente haría que el código "más alto", mientras que sigue siendo legible la UPC (Universal Product Code) de IBM fue seleccionada como la norma NAFC. IBM había diseñado cinco versiones de la simbología UPC para los requerimientos futuros del sector: UPC A, B, C, D y E
### 1974 NCR instaló un sistema de pruebas en el supermercado Marsh en Troy, Ohio. El 26 de junio de 1974, Clyde Dawson sacó un paquete de 10 gomas de Juicy Fruit de Wrigley de su cesta y fue escaneado por Sharon Buchanan a las 8:01 am. El paquete de chicle y el recibo están ahora en exhibición en la Institución Smithsonian . Fue la primera aparición comercial de la UPC.
### Expansión En 1978 menos del 1% de las tiendas de conveniencia en los Estados Unidos contaban con lectores. En 1981 el uso de los dispositivos creció a 10%, Tres años después ya representaba el 33% Hoy más del 60% de los negocios están equipados con esta tecnología.

### Simbologia
#### Simbología * Discreto: Espacio adiciional entre caracteres * Continuo: Caracteres sin espacio adicional * Dos anchos * Multiples anchos * Cerificación del código de barra: la calidad del código de barras en comparación con los estándares y especificaciones de la industria

Decodificación /Codificación

MSI: control de inventario, marcando contenedores de almacenamiento y estantes en entornos de almacén.

UPC-A

Consta de 12 digitos numéricos asignados exclusivamente a cada artículo comercial

Hay una correspondencia uno a uno entre los digitos y las líneas y espacios

UPC-E

Tiene 6 digitos

Es una versión de UPC con omisión de ceros (0)

Permite el uso de UPC en paquetes pequeños

EAN-13

Tiene 13 digitos

Utilizado en el comercio global

Permite identificar un tipo específico de producto en una configuración específica de paquete de un fabricante específico

EAN-8

Se deriva de EAN-13

Se introdujo para su uso en pequeños paquetes

### Matriciales Es una forma en dos dimensiones de representar información. Puede representar más datos por unidad de área No pueden ser leidos por un lector laser, sino por una basado en imagen

Azteca

Inventado en 1995 y publicado por AIM Inc. en 1997

Fue nombrado así por su similitud con una pirámide azteca

3832 dígitos, 3067 letras, 1914 bytes

Matriz de datos

Consiste en celdas blancas y negras organizadas en un patrón cuadrado o rectangular

El tamaño habitual de los datos es de uno pocos bytes hasta 1556 bytes

La longitud de los datos codificados depende del número de celdas en la matriz

Código QR

Quick Response code, "código de respuesta rápida"

Diseñado inicialmente para la industria automotriz japonesa

Se hizo popular por su rápida legibilidad y su mayor capacidad de almacenamieto

Fue inventado en 1994 por Denso Wave

## ZXing https://github.com/zxing/zxing

ZXing ("zebra crossing") es una librería Open Source para el procesamiento de imágenes de códigos de barras en formatos en una o dos dimensiones

  • El proyecto inició en Google en el año 2007
  • Está implementada en Java
  • Tiene varios ports a otros lenguajes
  • Distribuido bajo licencia Apache 2.0
  • Permite la generación y decodificación de códigos QR
### Formatos soportados |1D Product|1D Industrial|2D| |---|---|---| |UPC-A|Code-39|QR Code| |UPC-E|Code-93|Data Matrix| |EAN-8|Code-128|Aztec (beta)| |EAN-13|Codabar|PDF 417 (beta)| ||ITF|MaxiCode| ||RSS-14|| ||RSS-Expanded||

Google Play

Barcode Scanner

“Scan barcodes on products, or Data Matrix and QR Codes containing URLs, contact info, etc.”
  • Tiene una aplicación en la Google Play
  • Su calificación es de 4.1 basada en un total de 621.528
### Android Agregar a _dependencies_ en _build.gradle_ de _Module:app_ ``` dependencies { ... compile 'me.dm7.barcodescanner:zxing:1.9' ... } ``` Dar permisos en _AndroidManifest.xml_ ```xml <uses-permission android:name="android.permission.CAMERA" /> ```
Crear actividad para capturar stream de la cámara ```java public class ZXing extends AppCompatActivity implements ZXingScannerView.ResultHandler{ private ZXingScannerView mScannerView; @Override public void onCreate(Bundle state) { super.onCreate(state); mScannerView = new ZXingScannerView(this); setContentView(mScannerView); } @Override public void onResume() { super.onResume(); mScannerView.setResultHandler(this); mScannerView.startCamera(); } @Override public void onPause() { super.onPause(); mScannerView.stopCamera(); } @Override public void handleResult(Result result) { ZXingActivity.barcodeInfo.setText(result.getText()); onBackPressed(); } } ```
Crear el layout de la actividad que va a dar inicio a la captura ```xml <?xml version="1.0" encoding="utf-8"?> <RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android" android:layout_width="match_parent" android:layout_height="match_parent" android:padding="16dp"> <TextView android:id="@+id/zxing_result" android:layout_width="wrap_content" android:layout_height="wrap_content" android:layout_marginLeft="16dp" android:text="Nothing to read." android:textSize="20sp" android:layout_centerVertical="true" android:layout_centerHorizontal="true" /> <Button android:id="@+id/zxingScanButton" android:layout_width="wrap_content" android:layout_height="wrap_content" android:layout_above="@+id/zxing_result" android:layout_centerHorizontal="true" android:layout_marginBottom="17dp" android:text="Scan ZXing" android:onClick="scanZXing"/> </RelativeLayout> ```
Crear la actividad que use la actividad de captura ```java public class ZXingActivity extends AppCompatActivity{ public static TextView barcodeInfo; @Override protected void onCreate(Bundle savedInstanceState) { super.onCreate(savedInstanceState); setContentView(R.layout.activity_zxing); barcodeInfo = (TextView) findViewById(R.id.zxing_result); } public void scanZXing(View view){ startActivity(new Intent(this, ZXing.class)); } } ```
Instanciar la actividad ```java startActivity(new Intent(this, ZXing.class)); ```

## ZBar http://zbar.sourceforge.net/

ZBar es una suite de software Open Source para la lectura de códigos de barras desde varias fuentes tales como: video streams, imágenes, y lecturas de sensores

  • Está implementada en C
  • Existen aplicaciones para ser usados en Windows/Linux
Soporta varios tipos de códigos de barras * EAN-13 * UPC-A * UPC-E * EAN-8 * Code 128 * Code 39 * QR Code
### Android Agregar a _dependencies_ en _build.gradle_ de _Module:app_ ``` dependencies { ... compile 'me.dm7.barcodescanner:zbar:1.8.4' ... } ``` Dar permisos en _AndroidManifest.xml_ ```xml <uses-permission android:name="android.permission.CAMERA" /> ```
Crear actividad para capturar stream de la cámara ```java public class ZBar extends AppCompatActivity implements ZBarScannerView.ResultHandler { private ZBarScannerView mScannerView; @Override public void onCreate(Bundle state) { super.onCreate(state); mScannerView = new ZBarScannerView(this); setContentView(mScannerView); } @Override public void onResume() { super.onResume(); mScannerView.setResultHandler(this); mScannerView.startCamera(); } @Override public void onPause() { super.onPause(); mScannerView.stopCamera(); } @Override public void handleResult(Result result) { ZBarActivity.barcodeInfo.setText(result.getContents()); onBackPressed(); } } ```

Google Mobile Vision (Barcode API)

https://developers.google.com/vision/android/barcodes-overview
Provee un framework para encontrar objetos en fotos y videos. Este framework incluye detectores que localizan y describen objetos visuales en imágenes y videos. Actualmente incluye:
  • Rostros
  • Códigos de barras
  • Texto

La API de códigos de barras detecta códigos en tiempo real y en cualquier orientación

También puede detectar varios códigos de barras al mismo tiempo

### Formatos soportados |1D|2D| |---|---| |UPC-A|QR Code| |UPC-E|Data Matrix| |EAN-8|PDF-417| |EAN-13|Aztec| |Code-39|| |Code-93|| |Code-128|| |ITF|| |Codabar||
### Android Agregar a _dependencies_ en _build.gradle_ de _Module:app_ ``` dependencies { ... compile fileTree(dir: 'libs', include: ['*.jar']) compile 'com.google.android.gms:play-services:11.0.4' compile 'com.android.support:appcompat-v7:25.2.0' ... } ```
Definir el layout ```xml <?xml version="1.0" encoding="utf-8"?> <RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android" android:layout_width="match_parent" android:layout_height="match_parent" android:padding="16dp"> <SurfaceView android:id="@+id/camera_view" android:layout_width="640dp" android:layout_height="400dp" android:layout_alignParentLeft="true" /> <TextView android:id="@+id/mobile_vision_result" android:layout_width="wrap_content" android:layout_height="wrap_content" android:layout_below="@+id/camera_view" android:layout_centerHorizontal="true" android:layout_marginLeft="16dp" android:layout_marginTop="21dp" android:text="Nothing to read." android:textSize="20sp" /> </RelativeLayout> ```
Crear una actividad y en ella declarar ```java private SurfaceView cameraView; private CameraSource cameraSource; private BarcodeDetector barcodeDetector; ``` Sobrescribir el método _onCreate_ y obtener las referencias de los widgets definidos en el layout ```java cameraView = (SurfaceView) findViewById(R.id.camera_view); barcodeInfo = (TextView) findViewById(R.id.mobile_vision_result); ```
Crear el detector que se va a usar para detectar los códigos y la fuente de imágenes que se presentarán en el _SurfaceView_ ```java barcodeDetector = new BarcodeDetector.Builder(this).build(); cameraSource = new CameraSource .Builder(this, barcodeDetector) .setRequestedPreviewSize(640, 400) .build(); ``` Se puede cambiar la vista previa de la cámara con _setRequestedPreviewSize_
Agregar un _callback_ al _SurfaceHolder_ de la _SurfaceView_ que permita saber cuando se pueden dibujar la vista previa de la camára ```java cameraView.getHolder().addCallback(new SurfaceHolder.Callback() { @Override public void surfaceCreated(SurfaceHolder holder) { try { if (ActivityCompat.checkSelfPermission( getApplicationContext(), android.Manifest.permission.CAMERA) != PackageManager.PERMISSION_GRANTED) { return; } camerasource.start(cameraview.getholder()); } catch (ioexception ie) { log.e("camera source", ie.getmessage()); } } @override public void surfacechanged(surfaceholder holder, int format, int width, int height) {} @override public void surfacedestroyed(surfaceholder holder) { camerasource.stop(); } }); ```
Crear una instancia de una clase que implemente la interfaz _Detector.Processor_ para indicarle al detector que hacer cuando detecte un código ```java barcodeDetector.setProcessor(new Detector.Processor() { @Override public void release() {} @Override public void receiveDetections(Detector.Detections detections) { final SparseArray barcodes = detections.getDetectedItems(); if (barcodes.size() != 0) { barcodeInfo.post(new Runnable() { public void run() { barcodeInfo.setText(barcodes.valueAt(0).displayValue); } }); } } }); ```

Conclusiones

  • Cada librería tiene inconvenientes. Escoger la librería depende del proyecto
  • ZXing soporta más formatos de códigos que ZBar

ZXing

  • Es la más antigua de las 3, la más popular y aún recibe actualizaciones
  • Solo puede escanear códigos en la misma orientación de la cámara

ZBar

  • Es muy rápido para detectar los códigos, incluso antes de estabilizar el dispositivo
  • Detecta códigos en múltiples orientaciones
  • No se ha actualizado recientemente. El último commit fue en 2012
  • Carece de documentación
  • A veces se necesita reescanear

Google Mobile Vision

  • Puede detectar varios códigos al mismo tiempo
  • Detecta códigos en múltiples orientaciones
  • Es muy rápido para detectar los códigos
  • Siempre se tendrá la última versión de la librería
  • Es una librería nativa para android
  • Se debe descargar la librería antes de poder utilizarse
Si el dispositivo no tiene suficiente espacio no se descarga la librería
